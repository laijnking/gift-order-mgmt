# 供应商专属发货模板方案

## 1. 需求背景

礼品订单系统有三个角色：**客户**（下单方）→ **系统**（撮合订单）→ **供应商**（实际发货）。供应商发货时需要导出 Excel 发货通知单，但不同供应商对发货单格式和商品信息要求不同，之前系统只支持一套通用模板和字段规则。

### 原始问题

| 问题 | 描述 |
|------|------|
| 模板匹配不区分供应商 | 导出时不管选了几个供应商，都是用同一套模板，无法按供应商自动匹配 |
| 商品信息优先级单一 | 不管用哪个模板，商品字段统一取系统商品信息，供应商有自己的 SKU 体系无法体现 |
| 供应商选择体验差 | 设置模板关联供应商时用普通下拉，供应商一多难以定位 |

---

## 2. 改进方案

### 方案一：按供应商自动匹配模板

**改动文件**：`src/app/api/shipping-exports/batch/route.ts`

在供应商循环内调用 `resolvePreferredTemplate`，传入供应商 ID，系统自动匹配该供应商关联的模板。

匹配优先级：
1. 直接命中模板 `targetType=supplier` + `targetId` 一致
2. 命中 `template_links` 关联
3. fallback 到默认模板 / 第一个可用模板

```typescript
// 每个供应商循环内
if (!supplierTemplate) {
  const { template, source } = await resolvePreferredTemplate(client, {
    type: 'shipping',
    targetType: 'supplier',
    targetId: supplierId,
  });
  // ...
}
```

**效果**：每个供应商导出时自动拿到自己的发货模板，无需手动选择。

---

### 方案二：商品字段按模板类型决定优先级

**改动文件**：`src/app/api/shipping-exports/batch/route.ts`（`rowsForOrder` 函数）

根据是否为该供应商专属模板，决定商品信息来源：

| 场景 | productCode | productName | productSpec |
|------|-------------|-------------|-------------|
| 供应商专属模板 | `cu_product_code` → `product_code` | `cu_product_name` → `product_name` | `cu_product_spec` → `product_spec` |
| 通用模板 | `product_code` → `cu_product_code` | `product_name` → `cu_product_name` | `product_spec` → `cu_product_spec` |

说明：`cu_product_*` 为供应商商品映射信息，`product_*` 为系统商品信息。供应商专属模板优先输出供应商自己的 SKU，没有再用系统商品兜底。

---

### 方案三：模板设置页供应商选择改为可搜索

**改动文件**：`src/app/(app)/templates/page.tsx`

用 `Command` + `Popover` 可搜索组合框替换原来的普通下拉，支持输入供应商名称或编码实时过滤。

**效果**：供应商数量多时也能快速定位，不需要滚动长列表。

---

## 3. 数据链路总览

```
导出货源选择
    ↓
按供应商循环
    ↓
resolvePreferredTemplate(targetType='supplier', targetId=供应商ID)
    ↓
匹配到供应商专属模板？──否→ 通用模板 fallback
    ↓
isSupplierTemplate = true/false
    ↓
rowsForOrder → 按 isSupplierTemplate 确定商品字段优先级
    ↓
导出 Excel
```

## 4. 使用方式

### 设置模板

1. 进入"模板配置中心"
2. 点击"新增模板"
3. 模板类型选"发货通知模板"
4. 关联对象选"供应商"，用可搜索下拉选择具体供应商
5. 配置字段映射（可从预设方案快速应用，或导入 Excel 列名）
6. 保存

### 导出发货通知

1. 进入发货通知页面
2. 选择供应商（可多选）
3. 点击导出
4. 系统自动为每个供应商匹配其专属模板
5. 商品字段自动按供应商映射优先输出

## 5. 完整数据链路

```
Excel导入（客户商品编码）
    ↓
SKU匹配（映射到系统商品）→ 订单items（含 cu_product_* + product_*）
                                                    ↓
                           dispatchPendingOrder（扣库存）→ dispatch_records.items
                                                    ↓
                           rowsForOrder → 模板字段映射 → Excel导出
```

### 派发时的字段结构

`dispatchPendingOrder` 构建的 `dispatchItems` 包含两套商品编码：

| 字段 | 来源 | 用途 |
|------|------|------|
| `supplierProductCode` | 订单/SKU映射 → `cu_product_code` → `stock.product_code` | 供应商发货时使用 |
| `supplierProductName` | 订单/SKU映射 → `cu_product_name` → `stock.product_name` | 供应商发货时使用 |
| `supplierProductSpec` | 订单/SKU映射 → `cu_product_spec` → `stock.supplier_product_spec` | 供应商发货时使用 |
| `productCode` | `stock.product_code` | 内部成本核算 |
| `productName` | `stock.product_name` | 内部成本核算 |
| `productSpec` | `stock.supplier_product_spec` | 内部成本核算 |

```typescript
// dispatchPendingOrder 中的 dispatchItems 构建
dispatchItems.push({
  // 供应商商品编码（来自订单/SKU映射，供应商发货时使用）
  supplierProductCode: itemText(item, 'cu_product_code', 'cuProductCode') || stock.product_code,
  supplierProductName: itemText(item, 'cu_product_name', 'cuProductName') || stock.product_name,
  supplierProductSpec: itemText(item, 'cu_product_spec', 'cuProductSpec') || String(stock.supplier_product_spec || ''),
  // 系统商品编码（用于内部成本核算）
  productCode: stock.product_code,
  productName: stock.product_name,
  productSpec: stock.supplier_product_spec || '',
  quantity,
  unitCost,
  warehouseName: stock.warehouse_name || '',
});
```

### 导出时的字段优先级

| 场景 | productCode | productName | productSpec |
|------|-------------|-------------|-------------|
| 供应商专属模板（从 dispatchItems） | `supplierProductCode` → `productCode` | `supplierProductName` → `productName` | `supplierProductSpec` → `productSpec` |
| 通用模板（从 dispatchItems） | `productCode` → `supplierProductCode` | `productName` → `supplierProductName` | `productSpec` → `supplierProductSpec` |
| 供应商专属模板（无派发记录，走 parseItems） | `cu_product_code` → `product_code` | `cu_product_name` → `product_name` | `cu_product_spec` → `product_spec` |
| 通用模板（无派发记录，走 parseItems） | `product_code` → `cu_product_code` | `product_name` → `cu_product_name` | `product_spec` → `cu_product_spec` |

```typescript
// rowsForOrder 中的 context 构建
const supplierProductCode = (item as Record<string, unknown>).supplierProductCode as string || '';
const supplierProductName = (item as Record<string, unknown>).supplierProductName as string || '';
const supplierProductSpec = (item as Record<string, unknown>).supplierProductSpec as string || '';

const context: Record<string, unknown> = {
  // ...
  productCode: isSupplierTemplate
    ? (supplierProductCode || (item.productCode as string) || '')
    : ((item.productCode as string) || supplierProductCode || ''),
  productName: isSupplierTemplate
    ? (supplierProductName || (item.productName as string) || '')
    : ((item.productName as string) || supplierProductName || ''),
  productSpec: isSupplierTemplate
    ? (supplierProductSpec || (item.productSpec as string) || '')
    : ((item.productSpec as string) || supplierProductSpec || ''),
  // ...
};
```

## 7. 客户订单导入映射复用 + 客户反馈导出方案

### 7.1 需求背景

礼品订单系统存在一个闭环：

1. **客户**用自己习惯的 Excel 格式下单（列名可能是"货号"、"品名"、"规格"等）
2. **系统**将订单派发给供应商
3. **供应商**发货后回传快递信息
4. **系统**需要把发货结果反馈给**客户**

第 4 步的问题是：客户收到反馈单时，看到的是系统/供应商的 SKU（商品编码），而不是自己 Excel 里填的那个编码。客户无法将反馈单和自己的订单对应起来。

### 7.2 核心思路

> **客户用什么列名导入，系统就用什么列名导出。**
> 物流信息（快递公司、运单号）是系统追加的，不占用客户原始列。

### 7.3 完整数据流

```
客户上传Excel
    ↓
识别 customerCode
    ↓
计算 headerFingerprint
    ↓
查询 /api/column-mappings/history?customerCode=&fingerprint=
    ↓
命中？──否→ 用户手动映射 → 保存时指定导出列名 → 存入 feedback_export_headers
    ↓ 是
自动加载映射 → 折叠列表 + 显示"已加载"标记
    ↓
用户点"AI解析" → 订单提交
    ↓
supplier发货 → 回传快递单号
    ↓
导出客户反馈单 → 查询 feedback_export_headers → 用客户列名 + 物流列导出
```

### 7.4 数据库改动

**表：`column_mappings`**

新增一列：

```sql
ALTER TABLE column_mappings
ADD COLUMN IF NOT EXISTS feedback_export_headers JSONB DEFAULT '{}'::JSONB;
```

`feedback_export_headers` 格式为 `{ "客户列名": "系统字段名" }`，用于导出时还原客户原始列名。

### 7.5 导入时：保存反馈导出列名

**API：`POST /api/column-mappings`**

保存映射时，从 `columnMapping` 中构建"客户列名 → 系统字段名"映射，并支持用户覆盖导出列名：

```typescript
// 从 columnMapping 中构建：把列索引转成客户原始列名
const feedbackExportHeaders: Record<string, string> = {};
Object.entries(columnMapping).forEach(([colIndex, fieldName]) => {
  const header = sourceHeaders[parseInt(colIndex)];
  if (header && fieldName && fieldName !== 'ignore' && fieldName !== 'ext_keep') {
    feedbackExportHeaders[header] = fieldName;
  }
});

// 用户可额外指定导出列名覆盖（如"货号"导出时希望叫"商品编码"）
// 通过 body.feedbackExportHeaderOverrides 传入
```

### 7.6 导入时：自动加载历史映射

**前端：`src/app/(app)/order-parse/page.tsx`**

文件上传后，选择客户时自动查询历史映射：

```typescript
// 选择客户后，查询该客户的历史映射（最新一条活跃的）
if (selectedCustomer && excelPreview.length > 0) {
  const headers = excelPreview[headerRow];
  const fingerprint = buildHeaderFingerprint(headers);

  const res = await fetch(
    `/api/column-mappings/history?customerCode=${selectedCustomer}&fingerprint=${fingerprint}`
  );
  const data = await res.json();

  if (data.success && data.data?.length > 0) {
    const savedMapping = data.data[0];
    // 对比列名是否完全一致
    const savedHeaders = savedMapping.source_headers || [];
    if (JSON.stringify(headers) === JSON.stringify(savedHeaders)) {
      setColumnMapping(savedMapping.mapping_config || {});
      setMappingAutoLoaded(true);
      setMappingCollapsed(true);  // 折叠列表
    }
  }
}
```

### 7.8 导入时 UI：保存对话框中指定导出列名

每个字段映射行增加一个输入框：

```
货号       → [productCode ▼]    导出列名: [货号         ]
商品名称   → [productName ▼]    导出列名: [商品名称     ]
```

用户可以修改导出列名（默认预填 Excel 原始列名）。收集后传入 `feedbackExportHeaderOverrides`。

### 7.9 反馈导出：用客户列名 + 附加物流列

**API：`POST /api/export-feedback/batch`**

导出时读取客户的 `feedback_export_headers`：

```typescript
const { data: customerMapping } = await client
  .from('column_mappings')
  .select('feedback_export_headers, source_headers')
  .eq('customer_id', customerId)
  .eq('is_active', true)
  .order('created_at', { ascending: false })
  .limit(1)
  .maybeSingle();

const feedbackExportHeaders = customerMapping?.feedback_export_headers || {};

if (Object.keys(feedbackExportHeaders).length > 0) {
  // 用客户原始列名作为 Excel 表头
  const logisHeaders = ['快递公司', '运单号'];
  const allHeaders = [...Object.keys(feedbackExportHeaders), ...logisHeaders];

  // 每个 item 按 feedbackExportHeaders 的字段顺序取数据
  Object.entries(feedbackExportHeaders).forEach(([customerHeader, systemField]) => {
    context[customerHeader] = resolveFieldValue(item, systemField);
  });
  // 追加物流列
  context['快递公司'] = order.express_company || '';
  context['运单号'] = order.tracking_no || '';
} else {
  // 无历史映射：fallback 到默认逻辑 + 提示"首次导入后将自动保存映射"
}
```
