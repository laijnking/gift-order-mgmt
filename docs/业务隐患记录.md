## ⚠️ 商品档案编码包含中文及特殊字符

**发现时间**：2026-04-30
**影响系统**：礼品订单管理系统（正式库 `gift_order`）
**严重程度**：⚡ 中

### 问题描述

商品档案表 `products.code` 字段（商品编码）存在大量不符合规范的数据：

- **1828 条**商品编码包含中文、特殊字符或空格
- 占商品总数 8864 条的 **21%**

典型样本：
```
配件           通用配件      飞乐思柔薄暖宫带-经典版丽人版
3G3138*2       15WY1J(office)
3G3155*4       3G3191*2（新货号：3G6107)
```

### 潜在影响

1. **React Select 组件 key 重复** — 多个商品共用 `code = "配件"` 等名称，`key={p.code}` 导致 React 列表渲染异常或警告
2. **打印/导出文档乱码** — 发货单、标签等模板嵌 `product_code`，中文+特殊字符在某些字体下可能显示异常或被截断
3. **后续扩展风险** — 如果后续功能按 `code` 做 URL 参数、JSON key、正则匹配等，中文编码会产生兼容性问题

### 当前已确认安全

- ✅ SQL 用 Supabase 参数化查询，无注入风险
- ✅ 数据库 UTF-8 编码，中文存储正常
- ✅ 库存表 `stocks.product_code` 全部是标准格式，无中文/特殊字符
- ✅ 目前 `code` 未作为 URL 路径参数使用

### 建议处理方式

1. **短期**（影响可控）：对 Select 组件的 key 改用 `p.id` 替代 `p.code`，避免 key 重复
2. **中期**：业务侧确认新编码规则，给 1828 条问题商品重新生成规范编码（纯字母数字）
3. **长期**：在商品档案管理页面增加 `code` 字段格式校验，禁止输入中文/特殊字符

### 相关数据

```sql
-- 查看所有有问题的商品编码
SELECT code, name FROM products 
WHERE code ~ '[^a-zA-Z0-9_-]' 
ORDER BY updated_at DESC;

-- 按问题类型统计
SELECT 
  CASE 
    WHEN code ~ '[[:space:]]' THEN '含空格'
    WHEN code ~ '[a-zA-Z0-9]' AND code ~ '[^a-zA-Z0-9_-]' THEN '含特殊字符'
    ELSE '其他'
  END as type,
  COUNT(*) as cnt
FROM products
WHERE code ~ '[^a-zA-Z0-9_-]'
GROUP BY 1;
```

## 🔴 用户批量导入字段名错误（已修复）

**发现时间**：2026-04-30
**影响系统**：礼品订单管理系统 - 用户管理 - 用户导入
**严重程度**：🔴 高（数据一条都进不去）

### 问题描述

正式环境点击"导入"后无反应，数据未成功导入，页面无明确报错提示。

### 根因

`/api/users/batch` 接口字段名与数据库 `public.users` 表实际字段不匹配：

| 代码字段 | 实际字段 | 问题 |
|----------|----------|------|
| `password` | `password_hash` | 字段不存在，Supabase 静默忽略 |
| `status: 'inactive'/'active'` | `is_active: false/true` | 类型错误，插入失败 |

### 修复方案

文件：`src/app/api/users/batch/route.ts`
- `password` → `password_hash`
- `status: 'inactive'/'active'` → `is_active: false/true`

### 诊断方法

```sql
-- 查看 users 表实际字段
SELECT column_name, data_type FROM information_schema.columns
WHERE table_name='users' ORDER BY ordinal_position;
```
