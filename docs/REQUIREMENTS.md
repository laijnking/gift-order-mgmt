# 需求规格说明书

## 1. 项目背景

### 1.1 业务场景
礼品一件代发业务，客户下单后由供应商直接发货给终端客户，需要系统化管理订单、供应商、库存等资源。

### 1.2 目标用户
- **管理员**: 系统配置、用户管理、数据查看
- **业务员**: 客户开发、订单跟进
- **跟单员**: 订单录入、派发、回单处理

## 3. 非功能需求

### 3.1 性能需求
- 页面加载时间 < 3s
- API 响应时间 < 1s
- 支持 1000+ 订单数据展示

### 3.2 安全需求
- 用户认证和权限管理
- 数据隔离（按角色）
- SQL 注入防护

### 3.3 可用性需求
- 7x24 小时可用
- 数据定期备份
- 异常状态提示清晰

## 5. 业务流程

### 5.1 订单处理流程
```
1. 客户下单（微信/表格）
   ↓
2. 跟单员导入订单（AI/Excel）
   ↓
3. 订单进入"待派发"状态
   ↓
4. 系统匹配供应商（库存+价格+限制）
   ↓
5. 派发订单给供应商
   ↓
6. 供应商发货回传快递
   ↓
7. 跟单员导入回单
   ↓
8. 订单状态更新为"已回单"
   ↓
9. 反馈给客户
   ↓
10. 订单归档
```

### 5.2 库存预警流程
```
1. 库存变更（手动/订单扣减）
   ↓
2. 系统检查库存阈值
   ↓
3. 库存 ≤ 2 → 触发"尾货预警"
   ↓
4. 库存 = 0 → 触发"缺货警告"
   ↓
5. 创建预警记录
   ↓
6. 通知相关人员
```
