## 1. 系统概述

### 1.1 系统定位

礼品一件代发业务订单管理系统（Order Management System），支持多渠道订单收集、供应商智能匹配、订单自动拆分导出和快递回单管理。

### 1.2 核心业务流程

```
客户 ──(微信/表格)──▶ 跟单员 ──▶ 订单中心 ──▶ 匹配供应商 ──▶ 派发订单 ──▶ 供应商回传快递 ──▶ 汇总反馈
```

### 1.3 技术架构

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Next.js 16 (App Router) + React 19 |
| UI组件 | shadcn/ui (基于 Radix UI) |
| 样式方案 | Tailwind CSS 4 |
| 后端框架 | Next.js API Routes |
| 数据库 | Supabase PostgreSQL + RLS策略 |
| 文件解析 | XLSX (Excel解析) |

## 4. 订单录入模块

### 4.1 功能说明

订单录入模块支持通过Excel导入订单，自动解析表头、智能匹配商品和供应商，减少人工录入工作量。

### 4.2 功能清单

| 功能点 | 说明 | 优先级 |
|--------|------|--------|
| Excel上传 | 支持.xls/.xlsx格式，解析表头 | P0 |
| 列映射 | 自动检测列名，支持手动调整，可自动加载历史映射 | P0 |
| 历史映射复用 | 按表头指纹精确匹配，自动加载历史映射配置（可折叠） | P0 |
| 商品匹配 | 自动匹配系统商品（名称/规格/编码） | P0 |
| 供应商匹配 | 按库存、价格自动推荐供应商 | P0 |
| 尾货预警 | 库存≤2台时提醒，防止撞单 | P0 |
| 批量派发 | 一键派发订单到供应商 | P0 |
| 订单预览 | 解析后预览确认 | P0 |
| 错误提示 | 解析失败、匹配失败提示 | P1 |

### 4.3 实现逻辑

#### 4.3.1 Excel解析流程

```
1. 用户上传Excel文件
2. 系统读取Excel表头和数据行
3. 自动检测列名映射（支持以下字段）：
   - 基础信息：客户订单号、单据日期、单据编号
   - 商品信息：商品名称、商品编码、规格型号、数量、单价
   - 收货信息：收货人、收货电话、收货地址
   - 快递信息：物流公司、快递单号
4. 按订单号分组，生成订单列表
5. 返回解析结果供用户确认

历史映射自动加载：
- 选择客户后，系统根据Excel表头计算指纹（SHA256）
- 查询该客户是否有完全匹配的映射历史（按 header_fingerprint）
- 若命中，自动加载映射（绿色提示），默认折叠列映射区域
- 用户手动修改任一列时，自动加载标记取消
- 保存映射时，自动构建 feedback_export_headers 存入数据库
```

#### 4.3.2 商品匹配算法

```
优先级：规格匹配 > 编码匹配 > 条码匹配 > 名称匹配

1. 规格匹配：客户商品规格 精确匹配 系统商品规格
2. 编码匹配：客户商品编码 精确匹配 系统商品编码
3. 条码匹配：客户商品条码 精确匹配 系统商品条码
4. 名称匹配：客户商品名称 模糊匹配 系统商品名称

匹配结果：
- matched：精确匹配
- fuzzy：模糊匹配（需人工确认）
- unmatched：未匹配（需手动选择）
```

#### 4.3.3 供应商匹配规则

```
匹配条件：
1. 商品在供应商有库存（quantity > 0）
2. 商品与收货地址省份兼容（无快递限制）
3. 按综合成本排序（商品单价 + 预估运费）

优先级：
1. 内部仓库优先（type = 'self'）
2. 成本最优
3. 库存充足（库存 > 订单数量）
```

| Excel列名 | 系统字段 | 说明 |
|-----------|----------|------|

### 6.4 成本计算公式

```
总成本 = 商品单价 × 数量
总金额 = 总成本 + 运费 + 其他费用
利润率 = (售价 - 总金额) / 售价 × 100%
```

### 6.5 生命周期流转

```
1. 订单派发 → 创建成本记录（unit_cost, total_cost）
2. 回单导入 → 补全快递信息（express_company, tracking_no, returned_date）
3. 费用录入 → 补全运费和杂费（express_fee, other_fee, total_amount）
```

### 8.2 功能清单

### 8.3 回单匹配规则

```
匹配优先级：
1. 快递单号精确匹配
2. 订单号 + 收货人手机号匹配
3. 订单号 + 收货人姓名匹配
4. 收货人手机号 + 商品名称匹配

匹配成功 → 更新订单 tracking_no, express_company, returned_at
匹配失败 → 记录到待处理队列
```

| 1 | 订单录入 | 库存管理 | 订单录入时查询库存和供应商匹配 |
| 2 | 订单录入 | 历史成本库 | 派发订单时创建成本记录 |
| 3 | 订单录入 | 订单中心 | 提交订单后跳转到订单中心 |
| 4 | 订单中心 | 发货通知单 | 订单派发后生成发货通知单 |
| 5 | 发货通知单 | 物流回单 | 供应商发货后回传快递单号 |
| 6 | 物流回单 | 历史成本库 | 回单后更新成本记录快递信息 |
| 7 | 物流回单 | 订单中心 | 回单后更新订单状态为已回单 |
| 8 | 档案管理 | 订单录入 | 订单录入时关联客户、商品数据 |
| 9 | 档案管理 | 订单中心 | 订单详情显示客户、商品档案信息 |
| 10 | 档案管理 | 数据报表 | 报表数据来源于档案和订单数据 |

## 13. 数据字典

### 13.1 枚举值说明

#### 订单状态 (order_status)

| 值 | 标签 | 说明 |
|----|------|------|
| `pending` | 待派发 | 订单已导入，等待分配供应商 |
| `assigned` | 已派发 | 已分配供应商，等待发货 |
| `partial_returned` | 部分回单 | 部分商品已回单 |
| `returned` | 已回单 | 供应商已回传快递单号 |
| `completed` | 已完成 | 订单完结 |
| `cancelled` | 已取消 | 订单取消 |

#### 发货方类型 (shipper_type)

| 值 | 标签 | 说明 |
|----|------|------|
| `supplier` | 供应商 | 传统供应商，下载单据后发货 |
| `jd` | 京东 | 对接京东仓储 |
| `pdd` | 拼多多 | 对接拼多多仓储 |
| `self` | 自有仓 | 公司自有仓库 |
| `third_party` | 第三方 | 第三方代发 |

#### 发货方式 (send_type)

| 值 | 标签 | 说明 |
|----|------|------|
| `download` | 下载单据 | 导出Excel发给供应商发货 |
| `jd` | 京东API | API对接京东系统 |
| `pdd` | 拼多多API | API对接拼多多系统 |
| `self` | 自有发货 | 系统直接发货 |

#### 客户状态 (customer_status)

| 值 | 标签 | 说明 |
|----|------|------|
| `active` | 正常 | 正常合作 |
| `inactive` | 停用 | 暂停合作 |

#### 商品状态 (product_status)

| 值 | 标签 | 说明 |
|----|------|------|
| `active` | 在售 | 正常销售 |
| `inactive` | 停售 | 已下架 |
| `pre_order` | 预售 | 预售商品 |

#### 预警级别 (alert_level)

| 值 | 标签 | 说明 |
|----|------|------|
| `info` | 通知 | 普通提示 |
| `warning` | 警告 | 需要关注 |
| `error` | 错误 | 需要处理 |
| `critical` | 紧急 | 紧急处理 |

#### 匹配类型 (match_type)

| 值 | 标签 | 说明 |
|----|------|------|
| `exact` | 精确匹配 | 规格/编码完全一致 |
| `fuzzy` | 模糊匹配 | 名称相似 |
| `manual` | 手动匹配 | 人工选择 |

#### 库存变更类型 (change_type)

| 值 | 标签 | 说明 |
|----|------|------|
| `manual` | 手动调整 | 人工修改库存 |
| `import` | 批量导入 | Excel导入库存 |
| `order` | 订单扣减 | 订单派发扣减 |
| `adjust` | 系统调整 | 库存校正 |

#### 数据权限范围 (data_scope)

| 值 | 标签 | 说明 |
|----|------|------|
| `all` | 全部数据 | 可查看所有数据 |
| `department` | 本部门 | 仅查看本部门数据 |
| `self` | 仅本人 | 仅查看本人数据 |

### 13.2 常用业务术语

| 术语 | 说明 |
|------|------|
| 一件代发 | 客户下单后，由供应商直接发货给终端消费者 |
| 派发 | 将订单派发给供应商执行发货 |
| 回单 | 供应商发货后回传快递单号 |
| 撞单 | 两个订单使用了同一库存，导致发货冲突 |
| 尾货 | 库存数量很少的商品（通常≤2台） |
| SKU | 库存量单位，商品的最小库存单位 |
| 匹配码 | 唯一标识一次订单匹配的编码 |
