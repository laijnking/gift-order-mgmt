## 2. 详细交互说明

### 2.1 订单录入 → 档案管理

#### 2.1.1 订单录入 → 商品档案

**交互场景**：订单录入时，根据客户商品信息自动匹配系统商品

**交互流程**：
```
1. 用户上传Excel，包含客户商品名称/编码/规格
2. 系统查询 product_mappings 表，根据客户商品编码匹配系统商品
3. 如果未匹配到，根据商品名称/规格模糊匹配 products 表
4. 匹配成功后，返回：
   - systemProductId: 系统商品ID
   - systemProductName: 系统商品名称
   - systemProductSpec: 系统商品规格
   - systemProductCode: 系统商品编码
   - systemProductBrand: 系统商品品牌
5. 如果未匹配任何商品，提示用户手动选择或新增商品档案
```

**API调用**：
```
POST /api/match
Body: {
  customerCode: "C001",
  productName: "苏泊尔手持式吸尘器03S57-AP",
  productSpec: "03S57-AP",
  productCode: "KHD001"
}
```

**数据库查询**：
```sql
-- 精确匹配
SELECT * FROM product_mappings 
WHERE customer_id = $1 
AND (customer_product_code = $2 OR customer_product_spec = $3);

-- 模糊匹配
SELECT * FROM products 
WHERE name LIKE '%苏泊尔%' 
AND spec LIKE '%03S57%';
```

#### 2.1.2 订单录入 → 客户档案

**交互场景**：选择客户后，自动填充客户信息（业务员、跟单员）

**交互流程**：
```
1. 用户选择客户
2. 系统查询 customers 表，获取：
   - salesperson_id, salesperson_name: 业务员信息
   - order_taker_id, order_taker_name: 跟单员信息
3. 自动填充到订单表单
```

**API调用**：
```
GET /api/customers/[id]
```

### 2.2 订单录入 → 库存管理

#### 2.2.1 商品匹配后查询库存

**交互场景**：商品匹配成功后，查询该商品在各供应商的库存

**交互流程**：
```
1. 商品匹配成功后，获取 systemProductId
2. 查询 stocks 表，获取所有有库存的供应商
3. 按供应商分组，返回库存信息
4. 如果库存≤2台，触发尾货预警
```

      "stockPrice": 220.00,

      "supplierId": "uuid",

### 2.3 订单提交 → 订单中心

#### 2.3.1 订单提交

**交互流程**：
```
1. 用户确认订单信息，点击提交
2. 系统校验：
   - 必填字段：收货人、收货电话、收货地址
   - 商品信息：至少有一条商品明细
   - 供应商：至少选择一个供应商
3. 生成系统订单号：SYS-YYYYMMDD-XXXX-TIMESTAMP
4. 写入 orders 表
5. 写入 order_items 表
6. 返回提交结果
```

**API调用**：
```
POST /api/orders
Body: {
  sysOrderNo: "SYS-20260416-0001-M1ABC2D3",
  orderNo: "KHDD00000001",
  customerId: "uuid",
  receiverName: "杨红霞",
  receiverPhone: "18635565059",
  receiverAddress: "山西省长治市城区 解放西街36号",
  items: [
    {
      productId: "uuid",
      productName: "苏泊尔手持式吸尘器",
      quantity: 1,
      supplierId: "uuid"
    }
  ]
}
```

#### 2.3.2 订单号生成规则

**格式**：`SYS-{日期}-{序号}-{时间戳随机码}`

**示例**：`SYS-20260416-0001-M1ABC2D3`

**说明**：
- 日期：YYYYMMDD 格式
- 序号：当日订单从 0001 开始递增
- 时间戳随机码：确保高并发下唯一性

### 2.4 订单派发 → 发货通知单

#### 2.4.1 生成派发单

**触发时机**：订单状态从"待派发"变为"已派发"

**交互流程**：
```
1. 用户选择要派发的订单
2. 系统按供应商分组订单
3. 用户选择发货通知模板
4. 按模板配置生成 Excel 文件
5. 每个供应商生成一个 Sheet 或一个文件
```

**API调用**：
```
POST /api/export
Body: {
  orderIds: ["uuid1", "uuid2"],
  templateId: "uuid",
  groupBy: "supplier"
}
```

#### 2.4.2 扣减库存

**触发时机**：订单派发时

**交互流程**：
```
1. 订单派发后，更新 stocks 表
   - quantity = quantity - order_quantity
2. 写入 stock_versions 表记录变更
   - change_type: "order"
   - change_quantity: -order_quantity
3. 如果库存扣减到 ≤ 2 台，触发预警
```

**数据库操作**：
```sql
-- 扣减库存
UPDATE stocks 
SET quantity = quantity - $1, updated_at = NOW()
WHERE product_id = $2 AND supplier_id = $3;

-- 记录变更历史
INSERT INTO stock_versions (stock_id, before_quantity, after_quantity, change_quantity, change_type, created_at)
VALUES ($1, $4, $5, $6, 'order', NOW());
```

### 2.5 发货通知单 → 物流回单

#### 2.5.1 供应商回传

**交互场景**：供应商发货后，通过Excel回传快递单号

**交互流程**：
```
1. 用户上传回单Excel（包含订单号、快递公司、快递单号）
2. 系统按匹配规则匹配订单：
   - 优先级1：快递单号精确匹配
   - 优先级2：订单号 + 收货人手机号匹配
   - 优先级3：订单号 + 收货人姓名匹配
3. 匹配成功后更新 orders 表
4. 未匹配的记录进入待处理队列
```

**API调用**：
```
POST /api/returns
Body: {
  rows: [
    { orderNo: "KHDD00000001", expressCompany: "顺丰", trackingNo: "SF1234567890" },
    { orderNo: "KHDD00000002", expressCompany: "圆通", trackingNo: "YT9876543210" }
  ]
}
```

#### 2.5.2 订单状态更新

**匹配成功后**：
```sql
UPDATE orders SET
  express_company = $1,
  tracking_no = $2,
  returned_at = NOW(),
  status = 'returned',
  updated_at = NOW()
WHERE id = $3;
```

**写入回单记录**：
```sql
INSERT INTO return_records (order_id, express_company, tracking_no, matched_by, status, created_at)
VALUES ($1, $2, $3, 'auto', 'matched', NOW());
```

### 2.6 物流回单 → 历史成本库

#### 2.6.1 回单后更新成本信息

**交互场景**：回单导入后，更新历史成本库的快递信息

**交互流程**：
```
1. 回单匹配成功后
2. 查询 order_cost_history 表
3. 更新：
   - express_company: 快递公司
   - tracking_no: 快递单号
   - returned_date: 回单日期
4. 如果运费已录入，更新 total_amount
```

**数据库操作**：
```sql
UPDATE order_cost_history SET
  express_company = $1,
  tracking_no = $2,
  returned_date = CURRENT_DATE,
  total_amount = total_cost + express_fee + other_fee,
  updated_at = NOW()
WHERE order_id = $3;
```

### 2.6.4 导出记录 → 客户反馈导出（列名还原）

**交互场景**：供应商发货回传后，导出客户反馈单时，使用客户导入时的原始列名

**交互流程**：
```
1. 用户选择客户，触发导出反馈单
2. 系统查询 column_mappings 表，取 feedback_export_headers
3. 若存在映射：
   - Excel表头使用客户原始列名（来自 feedback_export_headers 的 key）
   - 商品字段值 cu_product_* 优先
   - 物流列（快递公司、运单号）追加在最后
4. 若不存在映射：fallback 到默认导出逻辑
5. 写入 export_records 表
```

**数据库操作**：
```sql
-- 查询客户列名映射
SELECT feedback_export_headers, mapping_config
FROM column_mappings
WHERE customer_code = $1 AND is_active = true
ORDER BY created_at DESC LIMIT 1;
```

**关键字段说明**：

| 场景 | 表头来源 | 商品字段优先级 | 物流列 |
|------|----------|--------------|--------|
| 有映射（本次新增） | 客户原始列名 | cu_product_* 优先 | 快递公司、运单号 |
| 无映射 | 默认字段名 | product_* | 快递公司、运单号 |

### 2.8 库存 → 预警中心

#### 2.8.1 尾货预警

**触发场景**：订单派发时检查库存

**触发条件**：库存 ≤ 2 台

**交互流程**：
```
1. 订单派发前查询库存
2. 如果库存 ≤ 2 台
3. 写入 alert_records 表
4. 预警类型：low_stock
5. 预警级别：warning
```

**数据库操作**：
```sql
INSERT INTO alert_records (id, rule_id, rule_code, stock_id, alert_type, alert_level, title, content, created_at)
VALUES ($1, $2, 'LOW_STOCK_ALERT', $3, 'low_stock', 'warning', '尾货预警', $4, NOW());
```

#### 2.8.2 预警规则配置

**规则类型**：

| type值 | 说明 | 配置示例 |
|--------|------|----------|
| `stock` | 库存预警 | `{"threshold": 2, "compare": "lte"}` |
| `order` | 订单预警 | `{"timeout_hours": 24}` |
| `price` | 价格预警 | `{"change_percent": 10}` |

#### 2.8.3 预警处理流程

```
1. 用户查看预警列表
2. 选择要处理的预警
3. 执行处理操作（补货/换供应商/忽略）
4. 标记为已处理
5. 记录处理人和处理时间
```

**数据库操作**：
```sql
UPDATE alert_records SET
  is_resolved = true,
  resolved_at = NOW(),
  resolved_by = $1,
  resolution = $2
WHERE id = $3;
```

## 3. 数据权限控制

### 3.1 权限范围说明

| 权限范围 | 说明 | 实现方式 |
|----------|------|----------|
| `all` | 查看所有数据 | 不做过滤 |
| `department` | 查看本部门数据 | 按 department 字段过滤 |
| `self` | 仅查看本人数据 | 按用户ID过滤 |

### 3.2 "仅本人"权限实现

**订单查询**：
```sql
-- 业务员：查看其负责客户的订单
SELECT * FROM orders o
JOIN customers c ON o.customer_id = c.id
WHERE c.salesperson_name = $1;

-- 跟单员：查看其跟单的订单
SELECT * FROM orders
WHERE operator_name = $1;
```

**前端传递**：
```
请求头：x-user-info: {"username": "张三", "role": "salesperson", "dataScope": "self"}
```

## 5. 异常处理

### 5.1 订单提交异常

| 异常场景 | 处理方式 |
|----------|----------|
| 订单号重复 | 使用时间戳+随机码重新生成 |
| 库存不足 | 提示用户选择其他供应商 |
| 客户不存在 | 提示先创建客户档案 |
| 商品未匹配 | 允许提交为"未匹配"状态 |

### 5.2 回单匹配异常

| 异常场景 | 处理方式 |
|----------|----------|
| 订单不存在 | 记录为"未匹配"，人工处理 |
| 快递单号重复 | 提示用户检查数据 |
| 订单已回单 | 跳过或提示重复回单 |

### 5.3 库存异常

| 异常场景 | 处理方式 |
|----------|----------|
| 库存为负数 | 禁止派发，提示补货 |
| 库存不足 | 提示库存不足或允许超卖 |
| 撞单 | 提示已有订单使用该库存 |
