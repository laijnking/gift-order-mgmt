# 导出交互验收清单

更新时间：2026-04-19

## 1. 目的

用于收口礼品订单管理系统中“发货通知导出 / 客户反馈导出 / 导出记录回看”相关页面的下载交互，作为后续无人值守推进中的固定回归标准。

覆盖页面：

- `/shipping-export`
- `/export-records`

覆盖对象：

- 整批 ZIP 下载
- 明细文件下载
- 已落盘文件
- 对象存储直链文件
- 未落盘、需重新生成的文件

## 2. 通用验收项

每次涉及导出交互调整时，至少检查以下项目：

- 页面展示的下载策略与真实行为一致
- 按钮文案与下载策略一致
- 下载提示语与页面语境一致
- 点击后按钮存在明确忙碌态，避免重复触发
- 触发后用户能收到明确反馈，不出现“点了但像没反应”
- 重新生成后的页面状态会自动刷新，不要求用户手动关弹窗再打开
- 单条明细重新生成不会污染整批导出记录
- 明细下载地址指向真实明细文件，而不是错误落到整批 ZIP

## 3. 分场景验收

### 3.1 已落盘本地文件

适用场景：

- `provider = local`
- `artifact.relative_path` 存在

验收项：

- 下载策略显示为 `本地下载`
- 按钮文案显示为 `下载`
- 提示语显示为 `将直接下载已落盘文件`
- 点击后直接下载，不经过重新生成

### 3.2 对象存储直链文件

适用场景：

- `provider = s3`
- `artifact.relative_path` 存在

验收项：

- 下载策略显示为 `直链下载`
- 按钮文案显示为 `直链下载`
- 提示语显示为 `将跳转到对象存储下载链接`
- 点击后走下载路由并跳转预签名链接

### 3.3 未落盘文件

适用场景：

- `artifact.relative_path` 不存在

验收项：

- 下载策略显示为 `按需重生成`
- 按钮文案显示为 `重新生成下载`
- 提示语与页面语境一致：
  - 导出记录页：`当前会先重新生成再下载`
  - 发货通知页：`当前会优先回退到整批ZIP`
- 点击后应触发重新生成或回退逻辑，而不是静默失败

## 4. 页面专项验收

### 4.1 发货通知导出页

页面：

- `/shipping-export`

验收项：

- 顶部整批 ZIP 区域展示模板、来源、存储、下载策略、提示语
- 每条供应商明细展示自己的下载策略和提示语
- 明细下载按钮存在独立 loading 态
- 整批 ZIP 下载按钮存在独立 loading 态
- 供应商行内“导出”不会因为状态异步导错对象

### 4.2 导出记录页

页面：

- `/export-records`

验收项：

- 列表页每条记录都提供统一下载入口
- 列表页下载按钮文案与真实行为一致
- 列表页下载按钮存在独立 loading 态
- 详情页顶部展示模板来源汇总、整批下载策略、整批提示语
- 明细行展示自己的下载策略、提示语和按钮状态
- 重新生成整批 ZIP 后，列表和详情自动刷新
- 重新生成单条明细后，详情和列表自动刷新，且只更新对应明细

## 5. 固定校验命令

每次涉及导出交互调整时，至少执行：

1. `corepack pnpm ts-check`
2. `corepack pnpm exec eslint 'src/lib/export-download.ts' 'src/app/(app)/export-records/page.tsx' 'src/app/(app)/shipping-export/page.tsx' --quiet`
3. `corepack pnpm fixtures:export-records`
4. `corepack pnpm fixtures:export-results`
5. `corepack pnpm fixtures:export-interactions`
6. `corepack pnpm check:export-ui`
7. `corepack pnpm check:business-smoke`
