# 导出文件存储部署说明

更新时间：2026-04-19

## 1. 当前支持的存储模式

系统当前支持两种导出文件存储模式：

1. `local`
   - 导出 ZIP 写入应用容器内的 `data/exports`
   - 适合单机 / 单容器部署
   - 当前 `docker-compose.yml` 已默认启用该模式，并通过 `app_data` 数据卷持久化

2. `s3`
   - 导出 ZIP 写入 S3 兼容对象存储
   - 适合多实例、跨容器、长期持久化场景
   - 当前代码已兼容 MinIO / S3 参数，但默认不启用

## 2. 本地磁盘模式

默认配置如下：

```env
EXPORT_ARTIFACT_PROVIDER=local
EXPORT_ARTIFACT_DIR=/app/data/exports
```

默认启动方式：

```bash
docker compose up -d --build
```

说明：

- `app` 服务会把导出文件写入 `/app/data/exports`
- `docker-compose.yml` 已挂载 `app_data:/app/data`
- 容器重建后，导出文件仍会保留在卷中

## 3. MinIO 对象存储模式

当前仓库已提供一个可选的 MinIO profile：

```bash
docker compose --profile object-storage up -d --build
```

建议配套环境变量：

```env
EXPORT_ARTIFACT_PROVIDER=s3
EXPORT_ARTIFACT_S3_BUCKET=gift-order-exports
EXPORT_ARTIFACT_S3_REGION=auto
EXPORT_ARTIFACT_S3_ENDPOINT=http://minio:9000
EXPORT_ARTIFACT_S3_ACCESS_KEY_ID=minioadmin
EXPORT_ARTIFACT_S3_SECRET_ACCESS_KEY=minioadmin
EXPORT_ARTIFACT_S3_FORCE_PATH_STYLE=true
EXPORT_ARTIFACT_SIGNED_URL_TTL=900
```

说明：

- `minio` 服务提供对象存储
- `minio-init` 会在启动时自动创建 bucket
- 应用只需要切换 `EXPORT_ARTIFACT_PROVIDER=s3`
- 下载路由会优先返回预签名 URL，默认有效期 `900` 秒

## 4. 推荐切换顺序

建议按这个顺序切换，不要一步跳太快：

1. 先在本地磁盘模式下验证导出、重生成、下载都正常
2. 再启用 MinIO profile
3. 把 `EXPORT_ARTIFACT_PROVIDER` 切到 `s3`
4. 重新导出一批发货通知和客户反馈，确认：
   - 新记录能下载
   - 老记录仍可按其原有 provider 下载

## 5. 当前边界

当前已完成：

- 导出记录会保存 `artifact.provider`
- 下载路由会按记录里的 provider 取文件
- 本地盘与对象存储两种后端都已在代码结构上支持

当前仍未完成：

- 明细级子文件的独立持久化
- 导出文件生命周期管理（清理、归档、过期策略）
