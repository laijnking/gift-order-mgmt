# 关键页面适配验收清单

更新时间：2026-04-19

## 1. 目的

用于收口礼品订单管理系统关键页面在手机、小尺寸笔记本和常规桌面宽度下的可用性问题，作为后续无人值守推进中的固定回归标准。

默认验收视口：

- 手机：`390 x 844`
- 小屏笔记本：`1024 x 768`
- 常规桌面：`1440 x 900`

## 2. 通用验收项

每个关键页面都应满足以下要求：

- 页面首屏没有被移动端菜单按钮、固定头部或顶部留白遮挡
- 页面主操作区在手机宽度下可见、可点、不会横向溢出
- 搜索区、筛选区和统计区在手机宽度下允许换行，不出现按钮被挤压到不可点击
- 表格页必须提供横向滚动兜底，列再多也不能把页面整体撑爆
- 弹窗宽度必须受视口约束，手机宽度下不贴边、不超出屏幕
- 双栏或多列表单在手机宽度下自动降为单列，在 `sm` 以上再展开
- 长文本、标签、徽标、分页条在小屏下不应遮挡相邻内容
- 页面在 `390 x 844` 下不存在明显的横向滚动条，除表格内容区外

## 3. 页面分组验收

### 3.1 全局布局

适用页面：

- `(app)` 下所有业务页

验收项：

- 移动端左上菜单按钮不遮挡页面标题
- 主内容区在移动端有稳定顶部留白
- 侧栏开合时不压住主要操作按钮

### 3.2 订单录入

页面：

- `/order-parse`

验收项：

- 主体在手机端为纵向布局，桌面端可回到双栏
- 客户选择、映射诊断、统计卡、动作按钮都可换行
- 明细编辑表单在手机端为单列
- 客户搜索弹层宽度不会超过视口

### 3.3 订单与导出链路

页面：

- `/orders`
- `/shipping-export`
- `/return-receipt`
- `/export-records`

验收项：

- 页头按钮在手机端换行后仍保留主操作优先级
- 预警侧栏、导出结果、手动匹配等弹层在手机端宽度安全
- 订单表、供应商表、回单表具备横向滚动兜底

### 3.4 系统设置与档案管理

页面：

- `/users`
- `/roles`
- `/alerts`
- `/templates`
- `/customers`
- `/products`
- `/sku-mappings`

验收项：

- 搜索、筛选、批量操作区在手机端纵向排列
- 列表表格不把页面整体撑出视口
- 新增、编辑、批量处理类弹窗在手机端可完整滚动浏览
- 多列字段区域在手机端降为单列

## 6. 回归执行方式

每次涉及页面结构修改时，至少执行以下动作：

1. 运行 `corepack pnpm ts-check`
2. 对本轮改动页面运行 `corepack pnpm exec eslint <page-files> --quiet`
3. 按本清单手工检查手机 / 小屏笔记本 / 桌面三档视口
4. 若发现新页面存在高密度表格或多列弹窗，先补入本清单再继续扩展
