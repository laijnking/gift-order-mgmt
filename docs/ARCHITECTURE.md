### 1.2 技术选型

| 组件 | 技术 | 理由 |
|------|------|------|
| 前端框架 | Next.js 16 | App Router、SSR、API Routes |
| UI 框架 | shadcn/ui | 基于 Radix UI、可定制 |
| 样式 | Tailwind CSS 4 | 原子化 CSS、快速开发 |
| 数据库 | Supabase | PostgreSQL、实时订阅、RLS |
| 状态管理 | React useState/useContext | 轻量级、够用 |
| 表单处理 | React Hook Form | 性能好、类型安全 |
| Excel 处理 | XLSX | 功能完整、浏览器兼容 |
| AI 集成 | LLM API | 订单智能解析 |

## 3. 安全设计

### 3.1 认证与授权

```typescript
// 角色定义
type Role = 'admin' | 'salesperson' | 'order_taker';

// 权限矩阵
const permissions = {
  admin: ['*'],                                    // 所有权限
  salesperson: ['orders:read', 'orders:create'], // 订单读写
  order_taker: ['*'],                              // 订单全权
};
```

### 3.2 RLS 策略

所有数据库表启用 Row Level Security，配置公开读写策略。

### 3.3 数据安全

- 用户密码加密存储
- API 请求参数校验
- SQL 注入防护（参数化查询）
- XSS 防护（React 自动转义）

## 4. 性能设计

### 4.1 前端性能

- 服务端渲染（SSR）提升首屏加载
- 图片懒加载
- 组件按需加载
- 防抖/节流优化

### 4.2 后端性能

- 数据库索引优化
- 分页查询
- 缓存策略（未来扩展）
