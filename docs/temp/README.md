## 二、收集顺序建议

### 2.1 推荐的收集顺序

```
第1步：整理商品档案
       ↓
第2步：整理客户档案（记录业务负责人员）
       ↓
第3步：整理发货方档案（供应商+仓库+京东/拼多多渠道）
       ↓
第4步：建立SKU映射表（商品关联）
       ↓
第5步：导入供应商库存
       ↓
第6步：配置系统用户
       ↓
第7步：开始导入历史/新订单
```

## 三、关键关联关系

### 3.1 数据关联图

```
                        ┌─────────────────┐
                        │    客户档案      │
                        │  (业务员/跟单员) │
                        └────────┬────────┘
                                 │
                                 ▼
┌─────────────┐          ┌─────────────────┐          ┌─────────────┐
│  发货方档案  │◄────────│    SKU映射表    │─────────►│  商品档案   │
│ (供应商+仓库)│          └─────────────────┘          └──────┬──────┘
└──────┬──────┘                                           │
       │                    ┌─────────────────┐            │
       ├───────────────────►│   供应商库存    │◄──────────┘
       │                    └────────┬────────┘
       │                             │
       ▼                             ▼
┌─────────────────┐          ┌─────────────────┐
│ 发货渠道配置     │          │    订单中心     │
│ (京东/拼多多)    │          │  (商品明细)     │
└─────────────────┘          └─────────────────┘
```

### 3.2 核心关联字段

| 关联关系 | 关联字段 | 说明 |
|----------|----------|------|
| 订单 → 客户 | customer_code | 订单关联客户档案 |
| 订单 → 商品 | 规格/编码/名称 | 订单商品关联商品档案 |
| SKU映射 → 商品 | product_code | 客户商品映射到系统商品 |
| 供应商库存 → 商品 | 规格/编码/名称 | 供应商商品关联系统商品 |
| 客户 → 业务员 | salesperson_id | 客户关联业务员 |
| 客户 → 跟单员 | operator_id | 客户关联跟单员 |

### 3.3 发货方档案类型说明

> **发货方档案 = 供应商 + 仓库 + 京东/拼多多渠道**，统一管理

| 类型 | 说明 | 发货方式 |
|------|------|----------|
| supplier | 普通供应商 | 下载发货 |
| jd | 京东一件代发 | 京东发货 |
| pdd | 拼多多一件代发 | 拼多多发货 |
| self | 自有仓库 | 自有发货 |
| third_party | 第三方仓库 | 下载发货 |

## 四、商品档案新增字段说明

### 4.1 尺寸与重量字段（用于运费计算）

| 字段 | 说明 | 示例 |
|------|------|------|
| 长度(cm) | 商品长度 | 30 |
| 宽度(cm) | 商品宽度 | 25 |
| 高度(cm) | 商品高度 | 20 |
| 重量(kg) | 商品重量 | 3.5 |

### 4.2 抛货判断规则

```
┌─────────────────────────────────────────────────────────────────┐
│  运费计算规则：                                                  │
│                                                                  │
│  体积重量 = 长 × 宽 × 高 ÷ 抛货系数（默认6000）                   │
│                                                                  │
│  实际重量 > 体积重量 → 按实际重量计费                             │
│  体积重量 > 实际重量 → 按体积重量计费（抛货）                      │
└─────────────────────────────────────────────────────────────────┘
```

## 五、常见问题

### 5.1 Q&A

**Q1: 发货方档案和供应商/仓库是什么关系？**
> A: 发货方档案是供应商和仓库的统一管理模块。包含了供应商、仓库、京东渠道、拼多多渠道等多种发货渠道类型。

**Q2: 如果客户商品和系统商品名称不一致怎么办？**
> A: 使用 SKU映射表 建立映射关系。在「SKU映射表数据收集模版」中填写：客户商品信息 → 系统商品信息

**Q3: 商品为什么要填写尺寸和重量？**
> A: 用于计算运费。如果不填写，系统将无法准确计算抛货商品的运费。

**Q4: 业务员和跟单员有什么区别？**
> A: 
> - 业务员：负责开发客户、录入订单
> - 跟单员：负责跟进订单、处理回单

### 5.2 注意事项

```
┌─────────────────────────────────────────────────────────────────┐
│  ⚠️ 重要提醒                                                     │
│                                                                  │
│  1. 所有编码字段必须唯一，不可重复                                │
│  2. 客户代码、发货方编码是系统关联的关键字段                       │
│  3. SKU映射表是实现自动匹配的核心，建议优先建立                   │
│  4. 供应商库存导入后，系统会自动尝试匹配系统商品                    │
│  5. 库存 ≤ 2 台时会触发尾货预警                                   │
│  6. 商品尺寸和重量用于运费计算，请尽量准确填写                     │
└─────────────────────────────────────────────────────────────────┘
```
