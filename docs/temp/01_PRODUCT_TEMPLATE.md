## 二、尺寸与重量说明

### 2.1 为什么要填写尺寸和重量？

```
┌─────────────────────────────────────────────────────────────────┐
│  运费计算规则：                                                  │
│                                                                  │
│  体积重量 = 长(cm) × 宽(cm) × 高(cm) ÷ 抛货系数                  │
│                                                                  │
│  实际重量 > 体积重量 → 按实际重量计费                             │
│  体积重量 > 实际重量 → 按体积重量计费（抛货）                     │
│                                                                  │
│  抛货系数：通常 5000 或 6000（快递公司标准）                      │
│  系统默认：6000                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 如何判断是否抛货？

| 场景 | 示例 | 计费方式 |
|------|------|----------|
| 轻抛货 | 毛绒玩具、被子 | 按体积重量计费 |
| 重货 | 铁锅、哑铃 | 按实际重量计费 |

### 2.3 填写示例

| 商品 | 长度 | 宽度 | 高度 | 体积重量 | 实际重量 | 判断 |
|------|------|------|------|----------|----------|------|
| 九阳电饭煲 4L | 30cm | 30cm | 25cm | 3.75kg | 3.5kg | 正常（按实际重量） |
| 毛绒玩具熊 | 40cm | 30cm | 40cm | 8kg | 0.5kg | 抛货（按体积重量） |
| 苏泊尔电压力锅 | 35cm | 35cm | 30cm | 6.1kg | 5kg | 正常（按实际重量） |

## 四、抛货商品识别

| 商品类型 | 特点 | 是否抛货 |

## 五、数据关联说明

```
┌─────────────────────────────────────────────────────────────────┐
│  商品档案关联关系：                                               │
│                                                                  │
│  1. 商品 → SKU映射表（通过商品编码关联客户商品）                   │
│  2. 商品 → 供应商库存（通过商品匹配关联）                          │
│  3. 商品 → 订单明细（通过商品信息关联）                           │
│  4. 商品尺寸/重量 → 运费计算                                      │
└─────────────────────────────────────────────────────────────────┘
```

## 六、填写注意事项

### 6.1 商品编码
- 必须唯一，不可重复
- 建议使用SKU编号或商品条码

### 6.2 尺寸和重量
- 所有尺寸单位为厘米(cm)
- 重量单位为千克(kg)
- 用于计算运费，请尽量准确填写

### 6.3 规格型号
- 填写商品的具体型号
- 系统会根据规格型号自动匹配供应商库存

### 6.4 生命周期状态

| 状态 | 说明 |
|------|------|
| 正常 | 正常销售商品 |
| 停产 | 已停止生产 |
| 清仓 | 库存清理中 |
