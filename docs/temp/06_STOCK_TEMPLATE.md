## 三、填写注意事项

### 3.1 供应商编码
- 必须与系统供应商档案中的编码一致
- 建议先导出供应商档案确认编码

### 3.2 商品名称和规格
- 填写供应商Excel或系统中的原始商品信息
- 系统会自动尝试匹配系统商品档案

### 3.3 库存数量
- 必须是**正整数**
- 为0表示无货
- ≤2台时会触发**尾货预警**

### 3.4 单价说明
- 填写供应商的**采购单价**
- 不含运费
- 用于成本核算

### 3.5 尾货预警规则

```
┌─────────────────────────────────────────┐
│  当库存数量 ≤ 2 时：                    │
│                                          │
│  1. 系统显示尾货预警标识                 │
│  2. 订单派发时提示库存不足               │
│  3. 建议及时补充库存或调整供应商          │
└─────────────────────────────────────────┘
```

## 四、库存更新说明

### 4.1 更新方式

| 方式 | 说明 | 适用场景 |
|------|------|----------|
| 全量覆盖 | 上传全部库存数据 | 定期同步 |
| 增量更新 | 只上传变化的数据 | 日常维护 |

### 4.2 更新规则

```
┌─────────────────────────────────────────────────────────┐
│  匹配条件（按优先级）：                                   │
│                                                           │
│  1. 按 规格型号 精确匹配                                  │
│  2. 按 商品编码 精确匹配                                  │
│  3. 按 商品名称 模糊匹配                                   │
│                                                           │
│  匹配成功 → 更新库存数量和单价                            │
│  匹配失败 → 新增库存记录（需后续关联系统商品）              │
└─────────────────────────────────────────────────────────┘
```

## 五、导入流程

### 5.1 导入步骤
```
1. 从供应商处获取最新库存Excel
2. 按本模版整理数据格式
3. 在「库存管理」页面导入
4. 系统自动匹配系统商品档案
5. 查看匹配统计报告
6. 对未匹配的商品进行人工处理
```

### 5.2 匹配结果说明

| 匹配类型 | 说明 | 处理方式 |
|----------|------|----------|
| 规格匹配 | 通过规格型号匹配 | 无需处理 |
| 编码匹配 | 通过商品编码匹配 | 无需处理 |
| 名称匹配 | 通过商品名称匹配 | 确认匹配正确性 |
| 未匹配 | 无法匹配系统商品 | 需添加SKU映射 |
