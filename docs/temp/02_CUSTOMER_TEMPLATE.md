## 三、填写注意事项

### 3.1 客户编码
- 编码应具有**唯一性**，不可重复
- 建议使用有规律的编号，如 `KH-001`、`KH-002`
- 编码确定后**不建议修改**

### 3.2 业务员和跟单员
- 必须是系统中已创建的用户
- 一个客户可以指定一个业务员和一个跟单员
- 业务员负责客户开发，跟单员负责订单跟进

### 3.3 数据权限说明
- **业务员**：只能查看自己负责的客户的订单
- **跟单员**：只能查看自己负责的客户的订单
- **管理员**：可以查看所有客户的订单

### 3.4 客户等级参考

| 等级 | 说明 | 特征 |
|------|------|------|
| A | 重点客户 | 订单量大、付款及时、长期合作 |
| B | 普通客户 | 订单量中等、稳定合作 |
| C | 小客户 | 订单量小、偶尔合作 |

## 四、批量导入说明

### 4.1 导入前检查
- ✅ 确保业务员、跟单员姓名与系统用户姓名一致
- ✅ 确保客户编码不重复
- ✅ 业务员和跟单员必须填写

### 4.2 导入流程
```
整理Excel → 上传到系统 → 系统自动校验 → 修正错误数据 → 确认导入 → 完成
```

### 4.3 常见错误

| 错误类型 | 原因 | 解决方法 |
|----------|------|----------|
| 业务员不存在 | 业务员姓名与系统不匹配 | 检查并修正业务员姓名 |
| 跟单员不存在 | 跟单员姓名与系统不匹配 | 检查并修正跟单员姓名 |
| 客户编码重复 | 同一编码出现多次 | 检查并修正客户编码 |
