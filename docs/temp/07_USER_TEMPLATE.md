## 三、角色说明

### 3.1 系统预设角色

| 角色 | 说明 | 权限范围 |
|------|------|----------|
| 管理员 | 系统管理员 | 所有功能、数据权限全部 |
| 业务员 | 负责客户开发和订单录入 | 仅本人数据 |
| 跟单员 | 负责订单跟进和回单处理 | 仅本人数据 |

### 3.2 自定义角色

管理员可根据业务需要创建自定义角色，配置具体的权限组合。

---

## 四、数据权限说明

### 4.1 权限类型

| 权限类型 | 说明 | 适用场景 |
|----------|------|----------|
| 全部 | 可查看所有数据 | 管理员、财务、主管 |
| 本部门 | 可查看本部门数据 | 部门负责人 |
| 仅本人 | 仅可查看本人数据 | 普通业务员、跟单员 |

### 4.2 权限生效范围

```
┌─────────────────────────────────────────────────────────────────┐
│  全部：可查看所有客户的订单                                       │
│          ↓                                                       │
│  本部门：可查看「业务员」或「跟单员」属于本部门的客户的订单         │
│          ↓                                                       │
│  仅本人：仅可查看「业务员」或「跟单员」等于自己姓名的客户的订单     │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 权限示例

| 用户角色 | 数据权限 | 可查看的订单 |
|----------|----------|--------------|
| 管理员 | 全部 | 所有客户的订单 |
| 业务员-李四 | 仅本人 | 业务员="李四"的客户的订单 |
| 跟单员-王五 | 仅本人 | 跟单员="王五"的客户的订单 |

## 五、填写注意事项

### 5.1 用户名规范
- 只能包含字母、数字、下划线
- 长度3-20个字符
- 必须唯一，不可重复
- 建议使用姓名拼音或英文名

### 5.2 初始密码
- 初始密码默认为：`123456`
- 用户首次登录后请及时修改密码

### 5.3 角色分配
- 根据用户的实际职责分配角色
- 避免给普通员工分配「管理员」角色

### 5.4 状态说明
- **启用**：可以正常登录和使用系统
- **禁用**：无法登录系统
