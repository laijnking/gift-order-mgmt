# 发货方档案数据收集模版

> 请业务人员按此模版整理发货方数据，用于系统初始化
>
> **发货方档案 = 供应商 + 仓库 + 京东/拼多多渠道**，统一管理所有发货渠道

## 二、发货类型说明

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| supplier | 供应商 | 普通供应商，手动下载订单发货 |
| jd | 京东渠道 | 京东一件代发 |
| pdd | 拼多多渠道 | 拼多多一件代发 |
| self | 自有仓 | 自有仓库发货 |
| third_party | 第三方仓 | 第三方仓库代发 |

## 五、数据关联说明

```
┌─────────────────────────────────────────────────────────────────┐
│  发货方档案关联关系：                                            │
│                                                                  │
│  1. 发货方 → 供应商库存（通过发货方ID关联）                       │
│  2. 发货方 → 订单派发（通过发货方选择关联）                        │
│  3. 京东/拼多多渠道 → 平台API对接（通过渠道ID关联）                 │

```

## 六、填写注意事项

### 6.1 编码规则
- 建议格式：`FHS-XXX`（FHS = 发货方首字母）
- 编码必须唯一，不可重复

### 6.2 类型选择
- 普通供应商选择 `supplier`
- 京东一件代发选择 `jd`
- 拼多多一件代发选择 `pdd`
- 自有仓库选择 `self`
- 第三方合作仓库选择 `third_party`

### 6.3 渠道配置
- 需要京东发货的发货方，需填写京东渠道ID
- 需要拼多多发货的发货方，需填写拼多多店铺ID

### 6.4 快递限制
- 设置后，该发货方的订单将不能使用指定的快递
- 多个快递用逗号分隔
