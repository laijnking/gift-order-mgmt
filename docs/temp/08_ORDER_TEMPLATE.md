## 三、填写注意事项

### 3.1 必填字段
- **订单号**：客户提供的原始订单号，不可重复
- **客户代码**：必须是系统已存在的客户编码
- **商品名称**：系统将自动匹配商品档案
- **数量**：正整数
- **收货人/收货电话/收货地址**：用于发货

### 3.2 客户代码说明
- 必须与系统客户档案中的编码一致
- 系统会根据客户代码自动关联：
  - 客户名称
  - 业务员
  - 跟单员

### 3.3 商品名称规范
- 填写客户Excel中实际使用的商品名称
- 系统会自动匹配系统商品档案
- 匹配规则：规格型号 → 商品编码 → 商品名称模糊匹配

### 3.4 收货地址规范
- 填写完整的收货地址
- 包含：省、市、区/县、详细地址
- 系统会根据地址提取省份信息用于运费计算

### 3.5 地址提取规则

```
收货地址示例：广东省深圳市南山区科技园
                    ↓
┌─────────────────────────────────┐
│  省份：广东省                    │
│  城市：深圳市                    │
│  区县：南山区                    │
└─────────────────────────────────┘
```

## 四、导入流程

### 4.1 导入步骤
```
1. 整理订单Excel
2. 在「AI订单录入」或「订单中心」页面导入
3. 系统自动检测表头并映射字段
4. 如有未识别字段，手动调整映射
5. 预览导入结果，确认无误后提交
6. 查看导入结果统计
```

### 4.2 字段自动检测
系统会自动检测以下字段名称：
- 订单号：`订单号`、`订单编号`、`销售订单编号`
- 商品名称：`商品名称`、`商品品名`、`品名`
- 规格型号：`规格`、`规格型号`、`型号`
- 收货人：`收货人`、`收件人`、`姓名`
- 收货电话：`电话`、`手机`、`收货电话`
- 收货地址：`地址`、`收货地址`

## 五、导入结果说明

### 5.1 成功结果
```json
{
  "success": true,
  "total": 10,
  "matchStats": {
    "total": 10,
    "bySpec": 5,      // 按规格匹配
    "byName": 3,      // 按名称匹配
    "byMapping": 1,   // 按SKU映射匹配
    "none": 1,       // 未匹配
    "matchRate": "90.0%"
  },
  "message": "成功导入 10 条订单，其中 9 条已匹配商品档案"
}
```

### 5.2 匹配类型说明

| 类型 | 说明 | 处理建议 |
|------|------|----------|
| 规格匹配 | 按规格型号精确匹配 | 最佳匹配，无需处理 |
| 编码匹配 | 按商品编码匹配 | 最佳匹配，无需处理 |
| 名称匹配 | 按商品名称模糊匹配 | 确认匹配正确性 |
| 未匹配 | 无法匹配系统商品 | 需添加SKU映射或商品档案 |
