# 历史成本库使用指南

> 本文档说明历史成本库的功能、数据流程和使用场景

---

## 一、功能概述

历史成本库记录每笔订单从派发到回单的全链路成本信息，用于：

| 功能 | 说明 |
|------|------|
| 成本核算 | 记录商品成本、运费、其他费用 |
| 利润分析 | 对比应收金额与实际成本 |
| 供应商分析 | 按供应商统计成本 |
| 财务对账 | 月度/季度成本报表 |

## 四、成本计算规则

### 4.1 成本计算公式

```
┌─────────────────────────────────────────────────────────────┐
│                                                              │
│  商品总成本 = 单台成本 × 数量                                 │
│                                                              │
│  总成本 = 商品总成本 + 快递费用 + 其他费用                     │
│                                                              │
│  总成本 = (unit_cost × quantity) + express_fee + other_fee   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 成本录入时机

| 阶段 | 自动写入 | 手动录入 |
|------|----------|----------|
| 订单派发 | ✅ 商品成本 | - |
| 回单录入 | ✅ 快递信息 | - |
| 成本管理 | - | ✅ 快递费用 |
| 成本管理 | - | ✅ 其他费用 |

## 五、API 接口

### 5.1 查询历史成本

```
GET /api/order-cost-history
```

**参数：**
- `search` - 搜索关键词（订单号/商品/客户）
- `supplierId` - 供应商ID
- `customerCode` - 客户代码
- `productCode` - 商品编码
- `startDate` - 开始日期
- `endDate` - 结束日期
- `page` - 页码
- `pageSize` - 每页数量

**响应：**
```json
{
  "success": true,
  "data": [...],
  "stats": {
    "totalQuantity": 100,
    "totalCost": 50000.00,
    "totalExpressFee": 1200.00,
    "totalOtherFee": 0,
    "totalAmount": 51200.00
  },
  "pagination": {...}
}
```

### 5.2 更新快递费用

```
PATCH /api/order-cost-history/fee
```

**请求体：**
```json
{
  "orderId": "xxx",      // 订单ID（可选）
  "orderNo": "xxx",      // 客户订单号（可选，与orderId二选一）
  "expressFee": 12.5,    // 快递费用（必填）
  "otherFee": 0,         // 其他费用（可选）
  "remark": "备注"        // 备注（可选）
}
```

**响应：**
```json
{
  "success": true,
  "message": "已更新 1 条记录的费用信息",
  "data": {
    "orderNos": ["ORD-xxx"],
    "expressFee": 12.5,
    "otherFee": 0,
    "updatedCount": 1
  }
}
```

### 5.3 批量导入

```
POST /api/order-cost-history/import
```

**请求体：**
```json
{
  "type": "orders",       // "orders" 或 "dateRange"
  "orderIds": ["id1", "id2"],  // 订单ID列表（type=orders时）
  "startDate": "2024-01-01",   // 开始日期（type=dateRange时）
  "endDate": "2024-01-31"      // 结束日期（type=dateRange时）
}
```

## 六、使用场景

### 6.1 场景一：月度成本统计

```
需求：查看某客户的月度成本报表

步骤：
1. 在历史成本库页面筛选
   - 客户代码：xxx
   - 日期范围：2024-01-01 至 2024-01-31

2. 查看统计数据
   - 总订单数
   - 总商品数量
   - 总商品成本
   - 总快递费用
   - 总成本

3. 导出报表
```

### 6.2 场景二：供应商成本对比

```
需求：对比不同供应商的价格

步骤：
1. 在历史成本库页面筛选
   - 日期范围：选择时间段

2. 按供应商汇总
   - 供应商A：平均成本 ¥95
   - 供应商B：平均成本 ¥92
   - 供应商C：平均成本 ¥90

3. 分析最优供应商
```

### 6.3 场景三：利润核算

```
需求：计算某订单的利润

步骤：
1. 查询订单
   - 订单号：ORD-20240101

2. 获取成本信息
   - 商品成本：¥95
   - 快递费用：¥12
   - 总成本：¥107

3. 获取收入信息
   - 应收金额：¥159

4. 计算利润
   - 利润 = 收入 - 成本 = ¥159 - ¥107 = ¥52
```

## 八、常见问题

### Q1：为什么历史成本库没有数据？

**可能原因：**
1. 尚未执行「导出派发单」操作
2. 订单尚未派发
3. 派发的订单尚未匹配到供应商

**解决方案：**
1. 在「发货通知单」页面执行导出派发单
2. 确认订单已成功派发

### Q2：如何补录历史订单的成本？

**方法一：通过页面手动录入**
1. 进入「历史成本库」页面
2. 点击「批量导入」
3. 选择日期范围或订单
4. 系统将按订单派发的逻辑写入成本

**方法二：手动创建记录**
1. 进入「历史成本库」页面
2. 点击「手动录入」
3. 填写成本信息

### Q3：总成本是如何计算的？

**公式：**
```
总成本 = 商品成本 + 快递费用 + 其他费用
       = (unit_cost × quantity) + express_fee + other_fee
```

**示例：**
- 单台成本：¥95
- 数量：2台
- 快递费用：¥12
- 其他费用：¥0

```
商品成本 = 95 × 2 = ¥190
总成本 = 190 + 12 + 0 = ¥202
```

### Q4：如何导出成本报表？

1. 进入「历史成本库」页面
2. 设置筛选条件
3. 点击「导出Excel」
4. 选择导出字段
5. 下载报表
