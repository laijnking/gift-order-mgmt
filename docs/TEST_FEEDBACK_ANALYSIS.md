## 三、核心业务流程问题

### 问题1：订单派发状态流程
**现状**：选定供应商后就进入了"已派发"状态  
**期望**：选定供应商只能算"待发货/待派发"，真正发给供应商才能算"已派发"（对应已导出发货单）

### 问题2：供应商选择效率
**现状**：逐单选择供应商，旺季效率低  
**期望**：使用统一的共同维护的供应商匹配表，公式一拉，先分配默认最佳选择

### 问题3：订单拆分
**现状**：没有拆分多数量订单的功能  
**期望**：支持将多数量订单拆分为多个小订单
