# 订单解析 - 操作指引

更新时间：2026-05-03

## 1. 功能说明

订单解析模块支持通过 Excel 批量导入订单，核心能力：
- 自动识别表头并建立列映射（SHA256 指纹匹配）
- 四级优先级智能匹配系统商品（规格→编码→条码→名称）
- 自动查询发货方库存并推荐最优发货方
- 历史映射一键复用，再次导入同类表格无需重复配置
- 客户反馈导出时按原始列名和顺序还原

---

## 2. 操作流程

### 2.1 首次导入

```
选择客户 → 上传 Excel → 手工配置列映射 → 解析匹配商品 → 确认发货方 → 提交订单
```

1. **选择客户** — 左侧面板选择客户，系统自动联动业务员/跟单员
2. **上传 Excel** — 支持 `.xls` / `.xlsx`，上传后选择要解析的 Sheet
3. **配置列映射** — 系统自动检测表头，首次需手工指定各列对应的系统字段
4. **解析订单** — 系统按四级优先级自动匹配商品，同时查询各发货方库存
5. **确认发货方** — 系统按规则推荐默认发货方，不合适可手工批量重选
6. **提交订单** — 提交时自动保存列映射配置（映射历史 +1）

### 2.2 再次导入（同客户同类表格）

```
选择客户 → 上传 Excel → 自动加载历史映射（绿色提示）→ 解析匹配 → 确认发货方 → 提交订单
```

与首次导入的区别：
- 上传 Excel 后，系统按表头指纹自动匹配历史映射，列映射区域默认折叠
- 仅当 Excel 列有变动（新增/删除/重命名列）时才需调整映射配置
- 其余流程与首次一致

### 2.3 商品匹配优先级

系统按以下优先级自动匹配商品：

| 优先级 | 匹配方式 | 说明 |
|--------|----------|------|
| 1 | 规格匹配 | 客户商品规格 精确匹配 系统商品规格 |
| 2 | 编码匹配 | 客户商品编码 精确匹配 系统商品编码 |
| 3 | 条码匹配 | 客户商品条码 精确匹配 系统商品条码 |
| 4 | 名称匹配 | 客户商品名称 模糊匹配 系统商品名称 |

匹配结果：
- **matched**：精确匹配
- **fuzzy**：模糊匹配（需人工确认）
- **unmatched**：未匹配（需手动选择）

### 2.4 发货方匹配规则

匹配条件：
1. 商品在发货方有库存（quantity > 0）
2. 商品与收货地址省份兼容（无快递限制）
3. 按综合成本排序（商品单价 + 预估运费）

优先级：
1. 内部仓库优先（type = 'self'）
2. 成本最优
3. 库存充足（库存 > 订单数量）

### 2.5 尾货预警

当库存 ≤ 2 台时，系统会显示尾货预警提示，防止撞单风险。

## 3. 列映射说明

### 3.1 生命周期

```
首次导入 → 手工配置映射 → 提交时自动保存（SHA256 指纹）
再次导入 → 指纹命中 → 自动加载（绿色提示，区域折叠）
列有变动 → 手动调整 → 提交时再次保存为新版本
```

映射自动保存，无需手动操作。「映射历史」按钮可查看和恢复历史版本。

### 3.2 客户反馈导出列名还原

提交时保存的映射配置包含客户的原始 Excel 列名和顺序。后续进行客户反馈导出时，系统按原始列名+顺序输出，并在末尾自动追加物流信息（快递公司、快递单号等），客户看到的列结构和他们提交的 Excel 一致。

### 3.3 支持的字段

| Excel列名 | 系统字段 | 说明 |
|-----------|----------|------|
| 商品名称 | product_name | 必填，客户商品名称 |
| 商品编码 | product_code | 选填，客户商品编码 |
| 规格型号 | product_spec | 选填，客户商品规格 |
| 数量 | quantity | 默认 1 |
| 单价 | price | 选填 |
| 收货人 | receiver_name | 必填 |
| 收货电话 | receiver_phone | 必填 |
| 收货地址 | receiver_address | 必填 |
| 客户订单号 | order_no | 选填，不填则自动生成 |
| 单据日期 | bill_date | 选填 |
| 备注 | remark | 选填 |

## 4. 数据流转

### 4.1 Excel 解析流程

```
1. 用户上传 Excel 文件
2. 系统读取整张 sheet 数据
3. 自动检测列名映射
4. 按订单号分组，生成订单列表
5. 返回解析结果供用户确认
```

### 4.2 重复导入识别

```
同批重复：同一批导入请求中，相同客户订单号只保留第一条
系统已存在：同客户同订单号在系统中已存在时跳过
结果弹窗展示：创建数量、跳过数量、重复原因、已存在系统单号
```

### 4.3 订单提交后数据流向

```
订单提交
    ↓
POST /api/orders
    ↓
orders 表 ← 订单主数据（items 为 JSONB 商品明细）
column_mappings 表 ← 列映射配置（提交时自动保存）
    ↓
跳转订单中心
```

## 5. 快捷操作

| 操作 | 说明 |
|------|------|
| 智能匹配发货方 | 为所有订单自动匹配最优发货方 |
| 批量选择/取消 | 全选或取消所有订单 |
| 复制订单 | 复制一条订单记录 |
| 新增订单 | 手动添加一条新订单 |
| 映射历史 | 查看和恢复历史映射配置 |

---

## 6. 注意事项

### 6.1 必填字段
- 商品名称
- 收货人
- 收货电话
- 收货地址

### 6.2 发货方选择
- 未选择发货方的订单将标记为「待派发」状态
- 可在订单中心后续手动派发

### 6.3 历史映射
- 映射按表头内容指纹匹配，不同表头结构需重新配置
- 每次提交会自动保存当前映射，无需手动操作
