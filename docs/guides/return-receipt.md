# 回单管理 - 操作指引

更新时间：2026-04-29

## 1. 功能说明

回单管理模块接收发货方回传的快递单号，支持直接上传 Excel 自动匹配订单，无需手动选择发货方。系统根据订单号自动匹配，并支持部分回单场景。

---

## 2. 操作流程

### 2.1 基本操作步骤

#### 步骤1：上传回单 Excel
1. 进入「回单管理」页面
2. 点击上传区域或拖拽回单 Excel 文件
3. 支持 `.xls` 和 `.xlsx` 格式

#### 步骤2：系统自动匹配
上传后，系统按以下优先级自动匹配订单：

| 优先级 | 匹配方式 | 说明 |
|--------|----------|------|
| 1 | 系统订单号 | 按 `sys_order_no` 精确匹配 |
| 2 | 客户订单号 | 按 `order_no` 精确匹配 |
| 3 | 模糊匹配 | 按订单号模糊匹配 |

#### 步骤3：处理匹配结果

**自动匹配成功**：
- 系统自动标记为「已匹配」
- 可直接进行批量确认

**冲突（多个候选）**：
- 进入「冲突」Tab
- 查看候选订单列表
- 手动选择正确订单

**未匹配**：
- 进入「待处理」Tab
- 检查数据是否有误
- 或手动关联订单

#### 步骤4：批量确认
1. 选择要确认的回单
2. 点击「批量确认」按钮
3. 系统更新订单物流信息

---

## 3. 回单匹配状态

| 状态 | 说明 | 处理方式 |
|------|------|----------|
| `pending` | 待匹配 | 系统自动匹配或手动处理 |
| `auto_matched` | 自动匹配成功 | 可直接确认 |
| `conflict` | 冲突（多个候选） | 手动选择正确订单 |
| `manual_matched` | 手动匹配成功 | 可直接确认 |
| `confirmed` | 已确认 | 回单完成 |

---

## 4. 部分回单支持

当同一订单分批导入回单时：

- 物流信息（快递公司、快递单号）用 `;` 隔开追加到订单
- 订单状态标记为 `partial_returned`（部分回单）
- 待全部商品回单后，状态更新为 `returned`（已回单）

---

## 5. 数据流转

### 5.1 回单匹配流程

```
用户上传回单 Excel
    ↓
系统自动匹配
    ↓
┌─────────────────────────────────────┐
│ 匹配结果判断                          │
│ 单一匹配 → auto_matched             │
│ 多候选匹配 → conflict (进入复核池)   │
│ 无匹配 → pending (进入复核池)        │
└─────────────────────────────────────┘
    ↓
用户批量确认
    ↓
更新 orders 表（tracking_no, express_company, freight_cost, returned_at）
    ↓
更新订单状态（partial_returned / returned）
```

### 5.2 回单确认后数据更新

```sql
UPDATE orders SET
  tracking_no = $1,        -- 快递单号
  express_company = $2,    -- 快递公司
  returned_at = NOW(),      -- 回单时间
  updated_at = NOW()
WHERE id = $3;

UPDATE order_cost_history SET
  express_company = $2,
  tracking_no = $1,
  returned_date = CURRENT_DATE,
  updated_at = NOW()
WHERE order_id = $3;
```

## 6. 冲突处理

### 6.1 冲突检测机制

当同一回单命中多个不同订单候选时，系统标记为冲突。

### 6.2 处理流程

1. 查看候选订单列表
2. 根据以下信息判断正确订单：
   - 收货人姓名
   - 收货电话
   - 商品信息
   - 收货地址
3. 选择正确订单
4. 点击「确认」

### 6.3 候选排序

系统按「更像当前回单的结果」优先排序候选订单。

## 8. 注意事项

### 8.1 Excel 格式要求
回单 Excel 应包含以下列（至少一列）：
- 订单号（系统订单号或客户订单号）
- 快递公司
- 快递单号
- 运费（可选）

### 8.2 匹配优先级
- 系统订单号 > 客户订单号
- 精确匹配 > 模糊匹配

### 8.3 重复回单
- 已回单的订单再次回单时，系统会追加快递信息
- 状态变为「部分回单」
