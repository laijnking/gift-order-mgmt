# 预警中心 - 操作指引

更新时间：2026-04-29

## 1. 功能说明

预警中心模块统一管理所有业务预警，包括库存不足预警、待派发超时预警、回单超时预警，支持预警规则配置和预警记录处理。

---

## 2. 预警类型

### 2.1 支持的预警类型

| 预警类型 | 触发条件 | 规则编码 | 说明 |
|----------|----------|----------|------|
| `low_stock` | 库存 ≤ 阈值（默认 2 台） | `LOW_STOCK_ALERT` | 发货方库存不足 |
| `order_timeout` | 待派发超过 N 小时 | `ORDER_TIMEOUT_ALERT` | 订单长时间未派发 |
| `return_delay` | 回单超过 N 小时未回传 | `RETURN_DELAY_ALERT` | 发货方长时间未回传 |

### 2.2 预警规则配置

#### 配置库存预警
1. 点击「规则配置」
2. 找到「库存不足预警」
3. 设置触发阈值（如：2 台）
4. 保存配置

#### 配置派发超时预警
1. 找到「待派发超时预警」
2. 设置超时时间（如：48 小时）
3. 保存配置

#### 配置回单超时预警
1. 找到「回单超时预警」
2. 设置超时时间（如：72 小时）
3. 保存配置

---

## 3. 操作流程

### 3.1 执行预警规则

#### 手动执行全部规则
1. 点击「执行全部规则」按钮
2. 系统扫描所有启用中的预警规则
3. 对满足条件的对象创建预警记录
4. 对已有预警的对象进行状态更新

#### 单独执行某条规则
1. 在规则列表中找到目标规则
2. 点击「执行」按钮
3. 系统执行该规则的扫描和预警创建

### 3.2 处理预警记录

#### 查看预警列表
1. 进入预警记录 Tab
2. 系统显示所有预警记录
3. 可按预警类型、状态等筛选

#### 标记已处理
1. 点击预警记录的「处理」按钮
2. 填写处理说明
3. 标记为已处理

### 3.3 启用/禁用规则

1. 在规则列表中找到目标规则
2. 点击开关按钮启用/禁用
3. 禁用后该规则不再触发预警

## 4. 预警执行器机制

### 4.1 执行逻辑

预警执行器（`/api/alert-rules/execute`）通过以下三阶段管理预警：

```
阶段1：新增
  条件：满足规则条件 + 无未处理预警
  操作：创建新 alert_records

阶段2：复用
  条件：满足规则条件 + 已有未处理预警
  操作：不做任何写操作（预警已存在）

阶段3：自动关闭
  条件：不满足规则条件 + 有未处理预警
  操作：更新 is_resolved=true, resolved_at=NOW()
```

### 4.2 执行流程图

```
扫描所有启用中的预警规则
    ↓
对每个规则扫描对应业务对象（stocks / orders）
    ↓
┌─────────────────────────────────────┐
│ 仍满足条件且无未处理预警 → 创建新预警  │
│ 仍满足条件且有未处理预警 → 复用既有预警 │
│ 不满足条件且有未处理预警 → 自动关闭预警  │
└─────────────────────────────────────┘
```

## 5. 预警记录表

| 字段 | 说明 |
|------|------|
| id | 预警记录ID |
| rule_id | 关联的预警规则ID |
| rule_type | 预警类型（low_stock/order_timeout/return_delay） |
| target_id | 触发对象ID（如：库存记录ID、订单ID） |
| target_type | 触发对象类型（stock/order） |
| alert_level | 预警级别（info/warning/critical） |
| title | 预警标题 |
| content | 预警内容 |
| is_resolved | 是否已处理 |
| resolved_at | 处理时间 |
| resolved_by | 处理人 |
| resolved_note | 处理说明 |
| created_at | 创建时间 |

## 6. 数据流转

### 6.1 预警创建流程

```
库存变更（派发/导入/调整）
    ↓
用户手动执行预警规则
    ↓
扫描库存记录（quantity ≤ threshold）
    ↓
检查是否已有未处理预警
    ↓
无预警 → 创建新预警记录
有预警 → 复用既有预警
```

### 6.2 预警自动关闭流程

```
库存补货（手动调整/导入）
    ↓
用户手动执行预警规则
    ↓
扫描库存记录（quantity > threshold）
    ↓
检查是否有未处理预警
    ↓
有预警 → 自动关闭（is_resolved=true）
```

## 8. 注意事项

### 8.1 预警阈值
- 库存预警默认阈值为 2 台
- 可根据实际情况调整

### 8.2 超时时间
- 派发超时和回单超时默认时间可配置
- 建议设置合理的时间范围，避免误报

### 8.3 自动关闭
- 当条件不再满足时，预警会自动关闭
- 无需手动处理所有预警

### 8.4 订单中心联动
- 预警记录会显示在订单中心侧栏
- 可直接在订单中心处理相关预警
