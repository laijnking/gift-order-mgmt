# 库存管理 - 操作指引

更新时间：2026-04-29

## 1. 功能说明

库存管理模块提供库存查询、版本历史追踪、价格历史记录功能，支持库存导入和预警规则配置。

---

## 2. 操作流程

### 2.1 库存查询

#### 基本查询
1. 进入「库存管理」页面
2. 系统默认显示所有库存记录
3. 可按以下条件筛选：
   - 商品名称/编码
   - 发货方
   - 仓库
   - 库存范围

#### 查看详情
1. 点击库存记录查看详情
2. 显示信息包括：
   - 商品基本信息
   - 各发货方库存数量
   - 最近更新时间

### 2.2 库存版本历史

1. 点击库存记录的「历史」按钮
2. 查看库存变更轨迹
3. 变更类型包括：
   - `import`：批量导入
   - `order`：订单派发扣减
   - `manual`：手动调整
   - `adjust`：系统调整
   - `return`：退货入库
   - `transfer`：调拨

### 2.3 价格历史

1. 点击库存记录的「价格」按钮
2. 查看价格调整历史
3. 显示每次价格变更的时间、新旧价格

### 2.4 库存导入

1. 点击「导入」按钮
2. 上传库存 Excel 文件
3. 系统解析并更新库存
4. 生成导入批次记录

---

## 3. 库存变更类型

| 类型值 | 说明 | 触发场景 |
|--------|------|----------|
| `import` | 批量导入 | Excel 导入库存 |
| `order` | 订单派发扣减 | 订单派发时扣减库存 |
| `manual` | 手动调整 | 人工修改库存 |
| `adjust` | 系统调整 | 库存校正 |
| `return` | 退货入库 | 订单退货入库 |
| `transfer` | 调拨 | 仓库调拨 |

---

## 4. 库存扣减规则

### 4.1 扣减时机

订单派发时（`dispatch_only` 或 `dispatch_with_persistence` 模式），系统自动扣减库存。

### 4.2 扣减流程

```
派发订单
    ↓
查询商品在各发货方的库存
    ↓
更新 stocks 表：quantity = quantity - order_quantity
    ↓
写入 stock_versions 表记录变更
    ↓
如库存 ≤ 2 台，触发尾货预警
```

---

## 5. 数据流转

### 5.1 库存查询流程

```
用户查询库存
    ↓
查询 stocks 表
    ↓
按商品和发货方分组
    ↓
返回库存列表
```

### 5.2 库存变更记录

```
库存变更触发
    ↓
写入 stock_versions 表：
- product_id
- supplier_id
- warehouse_id
- change_type
- change_quantity
- quantity_before
- quantity_after
- operator
- created_at
```

## 7. 注意事项

### 7.1 库存不足
- 库存不足时，系统会阻止派发
- 提示用户选择其他发货方或补充库存

### 7.2 撞单风险
- 当库存 ≤ 2 台时，系统显示尾货预警
- 提醒用户注意撞单风险

### 7.3 版本历史
- 所有库存变更都会记录版本历史
- 可追溯每笔库存的变更原因和操作人
