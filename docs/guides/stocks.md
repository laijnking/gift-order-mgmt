# 库存管理 - 操作指引

更新时间：2026-05-12

## 1. 功能说明

库存管理模块提供库存查询、版本历史追踪、价格历史记录功能，支持库存导入（含SKU映射匹配）和预警规则配置。

---

## 2. 操作流程

### 2.1 库存查询

#### 基本查询
1. 进入「库存管理」页面
2. 系统默认显示所有库存记录
3. 可按以下条件筛选：
   - 商品名称/编码
   - 发货方
   - 仓库
   - 库存范围

#### 查看详情
1. 点击库存记录查看详情
2. 显示信息包括：
   - 商品基本信息
   - 各发货方库存数量
   - 最近更新时间

### 2.2 库存版本历史

1. 点击库存记录的「历史」按钮
2. 查看库存变更轨迹
3. 变更类型包括：
   - `import`：批量导入
   - `order`：订单派发扣减
   - `manual`：手动调整
   - `adjust`：系统调整
   - `return`：退货入库
   - `transfer`：调拨

### 2.3 价格历史

1. 点击库存记录的「价格」按钮
2. 查看价格调整历史
3. 显示每次价格变更的时间、新旧价格

### 2.4 库存导入

#### 导入流程
1. 点击「导入」按钮
2. 上传库存 Excel 文件
3. 系统解析并匹配商品
4. 显示导入结果统计
5. 如有无法匹配的记录，可导出 CSV

#### 匹配优先级
系统按以下顺序匹配商品：

| 优先级 | 匹配方式 | 匹配字段 |
|--------|----------|----------|
| 1 | 商品编码 | products.code（精确匹配） |
| 2 | 规格型号 | products.spec（精确匹配） |
| 3 | 商品名称 | products.name（模糊匹配） |
| 4 | SKU映射 | product_mappings（按发货方商品编码） |
| 5 | 发货方商品编码 | products.spec 或 products.name（兜底） |

#### SKU映射匹配
当 Excel 中填写了「发货方商品编码」时，系统会优先查询 `product_mappings` 表：
- 条件：`supplier_id` + `supplier_product_code` 精确匹配
- 命中后验证 `product_id` 在 `products` 表中是否存在且启用

#### 无法匹配处理
当商品无法匹配时：
- 跳过该行，不写入库存
- 统计「无法匹配」数量
- 支持导出无法匹配的记录为 CSV

#### 导入字段说明

| Excel列名 | 系统字段 | 说明 |
|-----------|----------|------|
| 发货方编码 | supplierCode | 发货方编码 |
| 发货方名称 | supplierName | 发货方名称 |
| 发货方商品编码 | supplierProductCode | 发货方商品编码（用于SKU映射匹配） |
| 发货方商品名称 | supplierProductName | 发货方商品名称 |
| 发货方规格 | supplierProductSpec | 发货方商品规格 |
| 商品编码 | productCode | 系统商品编码 |
| 商品名称 | productName | 系统商品名称 |
| 规格型号 | productSpec | 系统商品规格 |
| 库存数量 | quantity | 库存数量 |
| 单价 | unitPrice | 单价 |
| 仓库 | warehouseName | 仓库名称 |
| 备注 | remark | 备注信息 |

## 3. 库存变更类型

| 类型值 | 说明 | 触发场景 |
|--------|------|----------|
| `import` | 批量导入 | Excel 导入库存 |
| `order` | 订单派发扣减 | 订单派发时扣减库存 |

| `adjust` | 系统调整 | 库存校正 |

| `transfer` | 调拨 | 仓库调拨 |

## 4. 库存扣减规则

### 4.1 扣减时机

订单派发时（`dispatch_only` 或 `dispatch_with_persistence` 模式），系统自动扣减库存。

### 4.2 扣减流程

```
派发订单
    ↓
查询商品在各发货方的库存
    ↓
更新 stocks 表：quantity = quantity - order_quantity
    ↓
写入 stock_versions 表记录变更
    ↓
如库存 ≤ 2 台，触发尾货预警
```

---

## 5. 数据流转

### 5.1 库存查询流程

```
用户查询库存
    ↓
查询 stocks 表
    ↓
按商品和发货方分组
    ↓
返回库存列表
```

### 5.2 库存变更记录

```
库存变更触发
    ↓
写入 stock_versions 表：
- product_id
- supplier_id
- warehouse_id
- change_type
- change_quantity
- quantity_before
- quantity_after
- operator
- created_at
```

## 7. 注意事项

### 7.1 库存不足
- 库存不足时，系统会阻止派发
- 提示用户选择其他发货方或补充库存

### 7.2 撞单风险
- 当库存 ≤ 2 台时，系统显示尾货预警
- 提醒用户注意撞单风险

### 7.3 版本历史
- 所有库存变更都会记录版本历史
- 可追溯每笔库存的变更原因和操作人
