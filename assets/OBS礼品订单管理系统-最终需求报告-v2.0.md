## 一、项目概述

### 1.1 业务背景

一件代发礼品业务（国内，无跨境）：

- **客户（下单方）** 提出礼品需求，指定型号/规格/数量/收货人信息
- **系统** 将订单派发给供应商/仓库，供应商/仓库完成采购或直接发货
- **供应商** 回传快递单号后，系统汇总并反馈给客户

### 1.2 核心痛点

| 痛点           | 说明                                                  |
| -------------- | ----------------------------------------------------- |
| 收单渠道多     | 微信群、在线表格、聚水潭系统，格式各异                |
| 表头不统一     | 客户发来的表头 ≠ 供应商要求的表头，需要映射          |
| 商品名称不一致 | 同一型号，客户/供应商/仓库各有不同叫法                |
| 订单匹配问题   | 同一客户多笔订单，需要唯一码确保一一对应              |
| 分次回单       | 当天订单可能分批回传快递信息，需要支持多次回单        |
| 供应商选择     | 多供应商/仓库，需要按成本/库存/时效/特殊要求自动匹配  |
| 订单拆分       | 每天需按各供应商的发单模板自动拆分成不同文件          |
| 运费与成本核算 | 各仓运费规则不一（重货/抛货），需结合单价核算综合成本 |
| 人工撞单风险   | 尾货（1-2台）派发时，跟单员容易出现并发抢单、撞单     |

### 1.3 完整业务流程

```
客户 ──(微信群/在线表格)──▶ 业务跟单员 ──▶ 订单总表
                                              │
                                              ▼
                                    按规则匹配供应商/仓库
                                              │
                                              ▼
                               派发给供应商（微信群/系统/聚水潭）
                                              │
                                              ▼
                               供应商回传快递单号（分批）
                                              │
                                              ▼
                               汇总快递信息
                                              │
                                              ▼
                               反馈给客户（群发/业务自取）
```

#### 4.3.2 供应商匹配算法

**匹配因素权重**：

| 因素       | 权重 | 说明                                             |
| ---------- | ---- | ------------------------------------------------ |
| 库存因素   | 50%  | 该供应商必须有商品库存（inventory.quantity > 0） |
| 自有仓权重 | 10%  | warehouse_type = 'OWN' 优先                      |
| 成本因素   | 20%  | 综合商品单价+预估运费                            |
| 快递限制   | 20%  | 排除不符合发货限制的供应商                       |

**省份匹配规则**：

- 收件地址是广东省 → 优先匹配广东省内仓库
- 仓库优先级：广东仓 > 正品仓 > 九阳仓 > 云海仓

### 4.13 订单预期预警规则

#### 4.13.1 模块说明

订单预期预警规则用于对订单流转过程中的关键节点进行时效监控，当订单未在规定时间内完成状态变更时，系统自动触发预警通知，提醒相关人员及时处理。

#### 4.13.2 预警规则定义

| 规则ID | 预警场景 | 触发条件 | 预警时间节点 | 说明 |
| ------ | -------- | -------- | ------------ | ---- |
| WARN-01 | 通知发货预警 | 从「订单录入」→「通知供应商发货」 | 当天 16:00 | 订单录入后至当天16:00未达「通知供应商发货」状态 |
| WARN-02 | 回单导入预警 | 从「通知供应商发货」→「导入回单」 | 次日 10:00 | 通知供应商发货后至次日10:00未更新状态 |
| WARN-03 | 客户反馈时效 | 从「待发货（订单录入）」→「订单反馈给客户」 | 24小时 | 订单录入后24小时内未反馈给客户 |

#### 4.13.3 预警规则配置

预警规则支持在系统设置中作为全局参数配置：

| 参数名称 | 默认值 | 说明 |
| -------- | ------ | ---- |
| warn_notify_cutoff_time | 16:00 | 通知发货预警时间点（当天） |
| warn_receipt_cutoff_time | 10:00 | 回单导入预警时间点（次日） |
| warn_customer_feedback_hours | 24 | 客户反馈时效（小时） |
| warn_enabled | true | 是否启用预警功能 |

#### 4.13.4 预警通知方式

| 通知方式 | 说明 |
| -------- | ---- |
| 系统消息 | 在订单中心列表中以颜色标记预警订单 |
| 站内提醒 | 推送提醒给订单责任人 |
