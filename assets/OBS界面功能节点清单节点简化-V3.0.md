## 17. 订单预期预警规则 (OrderWarningView) 🆕 v2.0新增

### 功能说明

订单预期预警规则用于对订单流转过程中的关键节点进行时效监控，当订单未在规定时间内完成状态变更时，系统自动触发预警通知。

### 预警规则定义

| 规则ID  | 预警场景     | 触发条件                                     | 预警时间节点 | 说明                                            |
| ------- | ------------ | -------------------------------------------- | ------------ | ----------------------------------------------- |
| WARN-01 | 通知发货预警 | 从「订单录入」→「通知供应商发货」           | 当天 16:00   | 订单录入后至当天16:00未达「通知供应商发货」状态 |
| WARN-02 | 回单导入预警 | 从「通知供应商发货」→「导入回单」           | 次日 10:00   | 通知供应商发货后至次日10:00未更新状态           |
| WARN-03 | 客户反馈时效 | 从「待发货（订单录入）」→「订单反馈给客户」 | 24小时       | 订单录入后24小时内未反馈给客户                  |

### 预警规则配置

| 参数名称                     | 默认值 | 说明                       |
| ---------------------------- | ------ | -------------------------- |
| warn_notify_cutoff_time      | 16:00  | 通知发货预警时间点（当天） |
| warn_receipt_cutoff_time     | 10:00  | 回单导入预警时间点（次日） |
| warn_customer_feedback_hours | 24     | 客户反馈时效（小时）       |
| warn_enabled                 | true   | 是否启用预警功能           |
